Especificación Funcional - Plataforma de Gestión de Limpiezas
Objetivo general
Desarrollar una plataforma web + app móvil para la gestión operativa de limpiezas, control de presencia y planificación de servicios asociados a reservas o solicitudes operativas adicionales.
La solución debe permitir:
* Configurar edificios y su estructura interna.
* Gestionar usuarios y permisos.
* Controlar fichajes de presencia.
* Gestionar tareas periódicas y tareas de check in/check out.
* Gestionar reservas y planificación automática de limpiezas.
* Asignar servicios a trabajadores.
* Permitir aceptación/rechazo de servicios desde móvil.
* Registrar ejecución real de tareas y tiempos.
* Obtener reportes operativos y analíticos.
* Visualizar operaciones en calendario y timelines.


Arquitectura general de módulos
La plataforma estará compuesta por los siguientes módulos:
1. Dashboard principal
2. Configuración
a) Edificios
b) Usuarios
3. Fichaje de presencia
4. Check In / Check Out
o Reservas
o Ordenes de trabajo


5. Tareas
6. Peticiones adicionales
7. Reportes


2a. Módulo de Configuración 2a.1 Configuración de Edificios Cada edificio deberá contener:
* ID de edificio
* Nombre
* Dirección completa
* Ciudad
* Provincia
Estructura jerárquica
La estructura será obligatoriamente:
Edificio
+-- Planta
+-- Zona
+-- Subzona (opcional)
Reglas
* Las zonas pertenecen a una planta.
* Las subzonas pertenecen a una zona.
* Las tareas pueden:
o estar directamente en la zona
o o dentro de subzonas
Restricción importante
Si una zona tiene subzonas:
* NO puede tener tareas directamente asociadas.
* Todas las tareas deberán estar en las subzonas.



2a.2 IDs internos
* Cada edificio tendrá un ID único.
* Cada zona tendrá un ID relacionado con el edificio.
* Las subzonas también tendrán ID únicos.


2a.3 Configuración de tareas
Las tareas se configurarán por zona o subzona.
Datos de una tarea
* Actividad (Tarea)
* Frecuencia
* Fecha de inicio
* Posibilidad de adjuntar fotos en ejecución
* Edificio/ID de edificio
* Zona/ID de zona
* Subzona/ID de subzona
Las tareas pueden configurarse directamente en la web o ser cargadas mediante Excel de carga masiva.


2a.4 Frecuencias disponibles
* Eventuales
* Diario (Lunes a Domingo)
* Lunes a Viernes
* Semanal
* Quincenal
* Mensual
* Trimestral
* Semestral
* Anual





2a.5 Comportamiento según frecuencia 
Tareas Eventuales
Aparecen únicamente dentro del módulo Tareas Eventuales cuando exista un servicio asociado.

Tareas periódicas
Aparecen dentro del módulo Tareas. Estados posibles:
* Programada
* En curso
* Completada
   * Vencida Ejemplo:
* Una tarea mensual estará como programada todo el mes en tanto y en cuanto no se realice, si no es realizada dentro del mes pasa automáticamente a "Vencida".


2a.6 QR por zona/subzona
Cada zona o subzona podrá generar un QR único. Al escanearlo desde la app:
* el trabajador accederá directamente a las tareas correspondientes.


2a.7 Importación masiva por Excel
El sistema debe permitir importar mediante Excel:
* edificios
* plantas
* zonas
* subzonas


* tareas
* frecuencias
   * Fecha de inicio Debe existir:
* descarga de plantilla vacía
* carga posterior del archivo completado


2a.8 Tareas con campos personalizados
* Se debe agregar la opción de configurar campos personalizados a las tareas de tal manera que pueda ańadirle un campo del tipo "Lista desplegable" en el que:
1. Se configure cada opción de la lista desplegable
2. Se pueda indicar si el campo es obligatorio o no
3. Se pueda indicar que dicho campo figure en el reporte final del servicio.
Comportamiento: al seleccionar la tarea para fichar se visualizará la lista desplegable
(con selección múltiple) en donde en caso de ser obligatorio haya que seleccionar una opción de la lista para poder ficharla.



2b. Módulo de Usuarios
El módulo "Trabajadores" pasa a llamarse:
Usuarios

* credenciales


2b.2 Roles de usuario


Administrador
Acceso total.


Encargado
Puede:
* ver todos los servicios aceptados
* iniciar cualquier servicio
* ver servicios del día
* fichar presencia sin seleccionar edificio
* Fichar tareas
Solo de edificios habilitados




Limpiador/a
Puede:
* fichar presencia dentro del edificio donde este trabajando
* Aceptar/rechazar servicios (tanto peticiones adicionales como check in/out)
* Iniciar ordenes de trabajo de check in/out o peticiones adicionales
* Fichar tareas
Solo de edificios habilitados


Cliente
Acceso limitado a:
* Reportes
* tareas
* Vista de ordenes de trabajo de Check In / out (Solo servicios aceptados, en curso o completados)
* Nueva petición de servicio


Solo de edificios habilitados.
No puede modificar información.


Proveedor
Acceso únicamente a:
* Vista de ordenes de trabajo de Check In/ out (Solo servicios aceptados, en curso o completados)
Sin permisos de modificación.


2b.3 Restricciones de fichaje
Cliente y Proveedor:
* NO fichan presencia.


2b.4 Asignación de edificios
Cada usuario tendrá:
   * edificios habilitados Relaciones bidireccionales:
* en usuario se ven edificios
* en edificio se ven usuarios

7. Módulo Tareas Eventuales
Este módulo tendrá dos submódulos:
1. Reservas


2. Ordenes de trabajo


7.1 Reservas
Las reservas pueden ingresar:
* vía API
* manualmente


Datos de reserva
* ID reserva
* edificio
* planta
* zona
* check in
* check out
* estado


7.2 Estados de reserva "Próxima"
Todos los días antes del check in el estado se definirá como "PROXIMA"


Check in
Se mostrará esta estado el día exacto de entrada. También puede mostrar:

En curso
Entre check in y día previo al check out.


Check out


El día exacto de salida.

Finalizada
Luego del check out.


7.3 Reglas importantes
Al lado de los estados"Proxima" y "Check in" además se mostrara:
* ? lista para habitar (Se ha registrado una limpieza anterior al check in)
* ? Zona no lista: Entre el último check out de esa misma zona y el dia actual, todavía no se ha registrado una limpieza. Aparecerá esta advertencia además si actualmente la zona esta con una reserva activa


7.4 Filtros reservas
Filtros por:
* edificio
* planta
* zona
* subzona
* fecha


8. Ordenes de trabajo
Cada reserva genera automáticamente:
* una planificación de limpieza para el día del check out. (Orden de trabajo)
* Cada orden de trabajo se generara sin ser asignada a nadie en particular y luego podrá ser asignada a un trabajador.
* Al clickar sobre la orden de trabajo debe mostrar de manera desplegable abajo, la reserva a la cual esa orden está asociada.


8.1 Estados de las ordenes de trabajo


* Sin asignar
* Asignada
* Aceptada
* En curso
* Completada
* Rechazada


8.2 Asignación de trabajadores
Solo puede asignarse a:
* limpiadores
* encargados


8.3 Recomendación automática
Al asignar:
   * recomendar trabajadores que ya tengan servicios ese día en el mismo edificio. Mostrar:
* nombre
* horario previo
* etiqueta "Recomendado"


8.4 Próximo check in
Durante asignación:
* mostrar el check in inmediato posterior del mismo espacio.


8.5 Rechazos
Si un trabajador rechaza:
* queda marcado en rojo
* puede reasignarse


Puede haber múltiples rechazos.

8.6 Orden visual
Todo debe ordenarse por:
* fecha
   * hora Opciones:
* ascendente
* descendente


8.7 Vista calendario
Debe existir una vista calendario mostrando:
* duración de reservas
* servicios de limpieza
   * trabajador asignado Filtros:
* edificio
* planta
* zona
* trabajador
* reservas
* servicios


8.8 Alertas visuales
Servicios:
* sin asignar deben parpadear en rojo.

* tareas vencidas
* presencia activa
* tiempos operativos


14. Consideraciones técnicas Persistencia de sesión
La sesión móvil debe mantenerse activa para:
* recibir solicitudes
* aceptar/rechazar servicios


Tiempo real
Actualizar en tiempo real:
* estados


* asignaciones
* aceptaciones
* fichajes


Conexión
La app debe ser capaz de funcionar sin conexión: al iniciar el servicio debe precargar todas las tareas y poder ficharlas aunque no tenga conexión. Luego al tomar cobertura la app sincronizará todos los datos recolectados.

* vistas tipo tabla

* timeline
* filtros avanzados


App móvil
Diseńo:
* simple
* rápido
* operativo
   * táctil Priorizando:
* velocidad

* claridad visual


16. Reglas operativas clave
* Un servicio solo puede completarse estando fichado.
* Encargado puede iniciar cualquier servicio. (aunque no se lo hayan asignado)


* Cliente y proveedor no fichan presencia.
* Las tareas vencen automáticamente.
* Las reservas generan automáticamente servicios.
* Los QR deben abrir directamente la zona correspondiente.
* Toda operación debe quedar registrada históricamente.
